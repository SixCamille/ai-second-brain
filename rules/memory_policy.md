# Memory Policy

Memory is intended to keep only durable information that has value for the user.

Before any creation or update, the agent should ask whether the information deserves to be remembered.

## Store

Prefer storing:

- durable preferences;
- projects;
- decisions;
- medium- or long-term tasks;
- routines;
- constraints;
- goals;
- useful resources;
- relations between objects;
- lessons learned;
- reasoning that led to a decision.

## Do Not Store

Avoid storing:

- general knowledge;
- public documentation;
- news with no personal or project impact;
- temporary conversations;
- details with no future value;
- unstable hypotheses;
- information already present in memory.

## Main Criterion

Information deserves to be stored when the user can reasonably benefit from finding it again later.

## Public Knowledge

Public facts usually should not become objects.

They may be stored when they directly explain a decision, project, preference, constraint, or lesson learned.

The stored object should then represent the user-specific context, not the public fact itself.
