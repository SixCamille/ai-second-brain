# Relations

Relations indicate that two objects are linked.

They do not describe the nature of the link, an action, or a hierarchy. That information belongs in the objects themselves and their content.

A relation should be created when it durably improves navigation or understanding of memory.

Do not create a relation simply because two objects are mentioned in the same conversation. The link must remain relevant beyond the immediate context.

## Importance

Importance only describes link strength.

It does not change the relation's meaning.

When no specific strength is identifiable, use the default value.

High importance should be reserved for strongly linked objects or objects whose understanding largely depends on each other.

## Good Practices

After every important creation or update, look for obvious relations with existing objects.

When creating a node, relations to explicitly mentioned durable entities are required, not optional. Search for each grounded linked person, organization, concept, resource, event, or reusable theme. If the linked node does not exist and deserves independent future retrieval, create it, then link it.

Avoid isolated objects when a logical link can be established without inventing information.

Conversely, do not create speculative or merely plausible relations.

The graph should reflect durable links, not only the flow of a conversation.

Only one relation can exist between two objects, regardless of direction.
